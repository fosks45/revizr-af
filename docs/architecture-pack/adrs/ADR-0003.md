# ADR-0003: Database Selection (PostgreSQL)

**Status:** Accepted
**Date:** 2026-05-15
**Feature:** 002-revizr
**Author:** solution-architect

---

## Context

Revizr requires persistent storage for: user and account data (C3/C7 classified),
session history and question attempt records, the questions database (30,000+ items
with rich metadata), diagnostic results and weakness maps, subscription records,
audit log (append-only), uploaded document metadata (deleted post-processing), and
notification preferences.

Key data access patterns driving the database choice:
1. **Question retrieval by topic / board / level / year:** The question pack assembly
   query must filter across multiple dimensions simultaneously. An index strategy on
   PostgreSQL `btree` indexes on `(board, level, subject, topic_id)` covers this pattern.
2. **Weakness map storage:** Per-user, per-subject weakness scores are semi-structured
   and may evolve as the schema matures. PostgreSQL JSONB provides flexibility without
   abandoning relational integrity.
3. **Audit log (Principle 10 / append-only):** A PostgreSQL table with an insert-only
   role and a trigger that raises an exception on UPDATE/DELETE enforces append-only
   semantics at the database layer.
4. **UK data residency (compliance C-012):** The database must be hosted on UK-resident
   infrastructure. PostgreSQL is available on AWS RDS in eu-west-2 (London), on Neon
   with a UK-resident primary, and on Supabase with a UK-region project.
5. **Full-text search on question content:** For the admin content management interface
   and future question recommendation, PostgreSQL `tsvector` / `tsquery` full-text
   search is adequate for v1 without an additional search service.

---

## Decision

**PostgreSQL 16 hosted on AWS RDS in eu-west-2 (London) with Multi-AZ enabled.**

Accessed from the backend via **Drizzle ORM** (TypeScript-native, schema-as-code,
migration tooling). Drizzle is selected over Prisma for this workload because:
- Drizzle generates SQL that is readable and auditable — each query is inspectable
  without Prisma's black-box query engine.
- Drizzle's TypeScript types are derived directly from the schema definition, not
  from a generated client — compile-time safety without an extra codegen step.
- Drizzle is lighter-weight at runtime (no query engine process) — important for
  Lambda/serverless deployment where cold-start time matters.

**Connection pooling:** PgBouncer in transaction mode via AWS RDS Proxy, ensuring
connection count stays within RDS limits even during peak concurrent session load.

**Encryption:** RDS encryption at rest using AWS KMS (customer-managed key) for C3+
and C7 data tables. All connections require SSL (`sslmode=require`). Column-level
encryption using `pgcrypto` for the most sensitive C3/C7 direct identifier fields
(student name, parent email) in addition to disk-level encryption.

**Backup:** RDS automated daily snapshots with 35-day retention. Point-in-time
recovery enabled. Snapshots replicated within eu-west-2 only (not cross-region,
per UK data residency constraint).

## Consequences

**Positive:**
- PostgreSQL JSONB enables flexible storage of diagnostic output (weakness scores per
  topic), which may have different numbers of topics depending on subject and exam level,
  without requiring schema migrations every time a new topic is added to a specification.
- `pg_trgm` extension provides trigram similarity search — sufficient for v1 question
  search in the admin interface without an external search service.
- Row-Level Security (RLS) in PostgreSQL can enforce data isolation at the database
  layer: a query for student data is rejected by RLS policy if the requesting session's
  `app.current_user_id` does not match the row's `owner_id`. This is a defence-in-depth
  layer supplementing application-layer authorisation checks.
- AWS RDS Multi-AZ provides automatic failover for the 99.5% availability SLO on the
  practice session flow without manual intervention.

**Negative / Trade-offs:**
- RDS Multi-AZ in eu-west-2 is more expensive than a single-AZ deployment. Cost is
  justified by the 99.5% availability SLO and the compliance requirement for UK-resident
  storage. Monthly cost at the expected v1 data scale: ~£150–200/month for db.t3.medium
  Multi-AZ.
- Drizzle is less mature than Prisma in the ecosystem. Mitigated by Drizzle's rapid
  development pace and the fact that Drizzle generates standard SQL — migration rollback
  is manually executable SQL, not dependent on ORM tooling.
- Column-level encryption (`pgcrypto`) adds query complexity for encrypted fields.
  Mitigated by encapsulating encrypted-field access in Drizzle helper functions that
  handle encrypt/decrypt transparently.

## Alternatives Considered

| Alternative | Reason Rejected |
|---|---|
| **MySQL / MariaDB** | MySQL lacks native JSONB (JSON column has no indexing comparable to PostgreSQL JSONB GIN indexes). MySQL full-text search is weaker than PostgreSQL `tsvector`. PostgreSQL RLS is more mature. Rejected on feature grounds. |
| **MongoDB** | MongoDB's document model is flexible, but: (a) no native relational joins — the question pack assembly query requires JOINs across users, topics, questions, and attempts; (b) MongoDB has weaker append-only audit log primitives; (c) ACID transaction support in MongoDB is newer and less battle-tested than PostgreSQL. Rejected. |
| **Supabase (PostgreSQL managed)** | Supabase is PostgreSQL-based and would be acceptable. Rejected in favour of AWS RDS because: Revizr's infrastructure is on AWS (Lambda, S3, SES); keeping the database in the same AWS account simplifies IAM, VPC peering, and CloudTrail audit logging. Supabase remains a viable fallback if AWS RDS proves operationally complex. |
| **PlanetScale (MySQL)** | PlanetScale's branching model is excellent for schema migrations. Rejected because: (a) MySQL limitations above; (b) PlanetScale's UK data residency guarantees are less established than AWS eu-west-2. |
| **DynamoDB** | DynamoDB's key-value access pattern is excellent for single-entity lookups but poor for the multi-dimensional question filtering queries required for pack assembly. Rejected as mismatched to the access patterns. |
